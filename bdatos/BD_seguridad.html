Base de Datos - Módulo Seguridad

Diseño y creación de la base de datos del sistema de cámaras y vigilancia

Diagrama de la Base de Datos

El diagrama muestra las tablas principales: Empleados, Roles, Cámaras, Grabaciones e Incidentes.

Descripción de Pertinencia de la Arquitectura

  • La arquitectura relacional permite relacionar empleados y roles con las cámaras y grabaciones.
  • Facilita consultas de incidentes, historial de grabaciones y reportes de seguridad.
  • Evita redundancia y mantiene la integridad de la información.

Normalización (1NF, 2NF, 3NF)

  • 1NF: Todos los campos son atómicos y cada registro es único.
  • 2NF: Cada atributo depende completamente de la clave primaria.
  • 3NF: No existen dependencias transitivas. Ej: tipo de cámara depende de la cámara, no del usuario.

Código SQL

-- Tabla Cámaras
CREATE TABLE Camaras (
    idCamara INT PRIMARY KEY AUTO_INCREMENT,
    ubicacion VARCHAR(100) NOT NULL,
    tipo VARCHAR(50),
    estado VARCHAR(20),
    fechaInstalacion DATE
);

-- Tabla Grabaciones
CREATE TABLE Grabaciones (
    idGrabacion INT PRIMARY KEY AUTO_INCREMENT,
    idCamara INT,
    fechaHoraInicio DATETIME,
    fechaHoraFin DATETIME,
    rutaArchivo VARCHAR(200),
    FOREIGN KEY (idCamara) REFERENCES Camaras(idCamara)
);

-- Tabla Incidentes
CREATE TABLE Incidentes (
    idIncidente INT PRIMARY KEY AUTO_INCREMENT,
    idEmpleado INT,
    idCamara INT,
    fechaHora DATETIME,
    descripcion TEXT,
    tipo VARCHAR(50),
    FOREIGN KEY (idEmpleado) REFERENCES Empleados(idEmpleado),
    FOREIGN KEY (idCamara) REFERENCES Camaras(idCamara)
);
