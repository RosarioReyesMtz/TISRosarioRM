Base de Datos - Módulo Estacionamiento

Diseño y creación de la base de datos del sistema de estacionamiento

Diagrama de la Base de Datos

El diagrama muestra las tablas principales: Espacios, Vehiculos, Reservas y RegistroEntradasSalidas.

Descripción de Pertinencia de la Arquitectura

  • La arquitectura relacional permite relacionar vehículos con espacios y reservas, así como registrar entradas y salidas de manera precisa.
  • Facilita consultas de disponibilidad y generación de reportes de uso y ocupación.
  • Evita redundancia y mantiene la integridad de la información.

Normalización (1NF, 2NF, 3NF)

  • 1NF: Todos los campos son atómicos y cada registro es único.
  • 2NF: Cada atributo depende completamente de la clave primaria.
  • 3NF: No existen dependencias transitivas. Por ejemplo, la información del vehículo depende de su registro, no de la reserva.

Código SQL

-- Tabla Espacios de Estacionamiento
CREATE TABLE Espacios (
    idEspacio INT PRIMARY KEY AUTO_INCREMENT,
    numeroEspacio VARCHAR(10) NOT NULL,
    estado VARCHAR(20), -- Disponible, Ocupado, Reservado
    tipo VARCHAR(50) -- Normal, VIP, Discapacitados
);

-- Tabla Vehículos
CREATE TABLE Vehiculos (
    idVehiculo INT PRIMARY KEY AUTO_INCREMENT,
    placa VARCHAR(20) NOT NULL,
    marca VARCHAR(50),
    modelo VARCHAR(50),
    color VARCHAR(20),
    idEmpleado INT, -- opcional: empleado asociado
    FOREIGN KEY (idEmpleado) REFERENCES Empleados(idEmpleado)
);

-- Tabla Reservas
CREATE TABLE Reservas (
    idReserva INT PRIMARY KEY AUTO_INCREMENT,
    idVehiculo INT,
    idEspacio INT,
    fecha DATE NOT NULL,
    horaInicio TIME,
    horaFin TIME,
    FOREIGN KEY (idVehiculo) REFERENCES Vehiculos(idVehiculo),
    FOREIGN KEY (idEspacio) REFERENCES Espacios(idEspacio)
);

-- Tabla Registro de Entradas y Salidas
CREATE TABLE RegistroEntradasSalidas (
    idRegistro INT PRIMARY KEY AUTO_INCREMENT,
    idVehiculo INT,
    idEspacio INT,
    fecha DATE NOT NULL,
    horaEntrada TIME,
    horaSalida TIME,
    FOREIGN KEY (idVehiculo) REFERENCES Vehiculos(idVehiculo),
    FOREIGN KEY (idEspacio) REFERENCES Espacios(idEspacio)
);
