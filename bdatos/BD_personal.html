Base de Datos - Módulo Personal

Diseño y creación de la base de datos del Hotel París

Diagrama de la Base de Datos

El diagrama muestra las tablas principales necesarias para gestionar el personal del hotel: Empleados, Roles, Asistencia y Horarios.

Descripción de Pertinencia de la Arquitectura de la Base de Datos

Se eligió una arquitectura relacional porque permite:

  • Organizar la información de los empleados de manera estructurada.
  • Establecer relaciones entre tablas, como Empleados ↔ Roles y Empleados ↔ Asistencia.
  • Facilitar consultas, reportes y mantenimiento de los datos.
  • Evitar redundancia y asegurar integridad de la información.

Normalización de la Base de Datos

Se aplicaron las primeras tres formas normales para garantizar integridad y eficiencia:

  • 1NF: Todos los campos contienen valores atómicos, no repetitivos y cada registro es único. Ej: La tabla Empleados tiene un campo “Nombre” indivisible.
  • 2NF: Todos los atributos dependen completamente de la clave primaria. Ej: Los datos de Roles no dependen parcialmente de la clave del empleado.
  • 3NF: No hay dependencia transitiva. Ej: Los salarios dependen del rol del empleado y no de otros datos no clave.

Código SQL

Creación de las tablas principales:

-- Tabla Roles
CREATE TABLE Roles (
    idRol INT PRIMARY KEY AUTO_INCREMENT,
    nombreRol VARCHAR(50) NOT NULL,
    descripcionRol VARCHAR(200)
);

-- Tabla Empleados
CREATE TABLE Empleados (
    idEmpleado INT PRIMARY KEY AUTO_INCREMENT,
    nombre VARCHAR(100) NOT NULL,
    puesto VARCHAR(50),
    salario DECIMAL(10,2),
    idRol INT,
    correo VARCHAR(100),
    telefono VARCHAR(20),
    fechaIngreso DATE,
    FOREIGN KEY (idRol) REFERENCES Roles(idRol)
);

-- Tabla Asistencia
CREATE TABLE Asistencia (
    idAsistencia INT PRIMARY KEY AUTO_INCREMENT,
    idEmpleado INT,
    fecha DATE NOT NULL,
    horaEntrada TIME,
    horaSalida TIME,
    FOREIGN KEY (idEmpleado) REFERENCES Empleados(idEmpleado)
);

-- Tabla Horarios
CREATE TABLE Horarios (
    idHorario INT PRIMARY KEY AUTO_INCREMENT,
    idEmpleado INT,
    diaSemana VARCHAR(15),
    horaInicio TIME,
    horaFin TIME,
    FOREIGN KEY (idEmpleado) REFERENCES Empleados(idEmpleado)
);
