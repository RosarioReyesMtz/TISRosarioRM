Base de Datos - Módulo Personal

Diseño y creación de la base de datos del Hotel París

Diagrama de la Base de Datos

Descripción de Pertinencia de la Arquitectura de la Base de Datos

La arquitectura de la base de datos está basada en el modelo relacional, lo cual permite

  • Integridad referencial entre módulos (ejemplo: personal vinculado a reservas, seguridad).
  • Escalabilidad para integrar nuevos cargos, usuarios y funcionalidades.
  • Seguridad de datos mediante roles y privilegios de acceso.
  • Optimización en consultas de nómina, reportes y gestión de personal.

Normalización de la Base de Datos

La tabla de Personal cumple hasta la 3FN (Tercera Forma Normal):

  • 1NF: Todos los atributos son atómicos (ejemplo: nombre, apellido, cargo).
  • 2NF: La tabla tiene clave primaria simple (id_personal), no hay dependencias parciales.
  • 3NF: No existen dependencias transitivas (ejemplo: salario depende solo de la persona, no de otro atributo).

Código SQL

Creación de las tablas principales:

-- Tabla de roles
CREATE TABLE Rol (
    id_rol INT PRIMARY KEY IDENTITY(1,1),
    nombre_rol VARCHAR(50) UNIQUE NOT NULL,
    descripcion VARCHAR(200)
);

-- Tabla de personal
CREATE TABLE Personal (
    id_personal INT PRIMARY KEY IDENTITY(1,1),
    nombre VARCHAR(100) NOT NULL,
    apellido VARCHAR(100) NOT NULL,
    dni VARCHAR(20) UNIQUE NOT NULL,
    telefono VARCHAR(20),
    correo VARCHAR(100) UNIQUE,
    direccion VARCHAR(200),
    id_rol INT NOT NULL,
    fecha_contratacion DATE NOT NULL,
    salario DECIMAL(10,2) NOT NULL,
    turno VARCHAR(20) NOT NULL CHECK (turno IN ('Mañana','Tarde','Noche')),
    estado VARCHAR(20) DEFAULT 'Activo' CHECK (estado IN ('Activo','Inactivo')),
    FOREIGN KEY (id_rol) REFERENCES Rol(id_rol)
);
