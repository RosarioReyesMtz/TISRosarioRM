Base de Datos - Módulo Restaurante

Diseño y creación de la base de datos del sistema de restaurante

Diagrama de la Base de Datos

Descripción de Pertinencia de la Arquitectura

  • Permite relacionar mesas, reservas, pedidos y platillos de forma eficiente.
  • Evita redundancia de información y facilita la generación de reportes de ventas.
  • Controla la relación entre meseros, cocineros y clientes.

Normalización (1NF, 2NF, 3NF)

  • 1NF: Todos los campos son atómicos y cada registro es único.
  • 2NF: Cada atributo depende completamente de la clave primaria.
  • 3NF: No existen dependencias transitivas; por ejemplo, la información de platillos depende solo del registro de platillo, no del pedido.

Código SQL

-- Tabla Mesas
CREATE TABLE Mesas (
    idMesa INT PRIMARY KEY AUTO_INCREMENT,
    numeroMesa VARCHAR(10) NOT NULL,
    capacidad INT NOT NULL,
    estado VARCHAR(20) -- Libre, Ocupada, Reservada
);

-- Tabla Platillos
CREATE TABLE Platillos (
    idPlatillo INT PRIMARY KEY AUTO_INCREMENT,
    nombre VARCHAR(100) NOT NULL,
    categoria VARCHAR(50),
    precio DECIMAL(10,2)
);

-- Tabla Reservas
CREATE TABLE ReservasRestaurante (
    idReserva INT PRIMARY KEY AUTO_INCREMENT,
    idMesa INT,
    nombreCliente VARCHAR(100),
    fecha DATE,
    hora TIME,
    FOREIGN KEY (idMesa) REFERENCES Mesas(idMesa)
);

-- Tabla Pedidos a Habitación
CREATE TABLE PedidosHabitacion (
    idPedidoHabitacion INT PRIMARY KEY AUTO_INCREMENT,
    numeroHabitacion VARCHAR(10) NOT NULL,
    idMesero INT,
    fecha DATE,
    hora TIME,
    estado VARCHAR(20), -- En preparación, Entregado
    FOREIGN KEY (idMesero) REFERENCES Empleados(idEmpleado)
);

-- Tabla Detalle de Pedidos a Habitación
CREATE TABLE DetallePedidosHabitacion (
    idDetalle INT PRIMARY KEY AUTO_INCREMENT,
    idPedidoHabitacion INT,
    idPlatillo INT,
    cantidad INT,
    FOREIGN KEY (idPedidoHabitacion) REFERENCES PedidosHabitacion(idPedidoHabitacion),
    FOREIGN KEY (idPlatillo) REFERENCES Platillos(idPlatillo)
);
